# Format

```md
# Goal

1 sentance goal

# In Scope

- list of inscope requirements

# Out of Scope

- List of out of scope requirements

# Acceptance Criteria

- List of criteria need for this to have reached the goal

# Implementation Plan

A plan with some code snippet of only the major changes that will be added leave out minor changes

# Anything else (optional)

```

# Folder Structure

- `specs/wip/`: draft specs being actively written/refined.
- `specs/ready/`: approved specs ready for implementation.
- `specs/done/`: completed specs with implementation finished.

# Workflow

- Create or edit draft specs in `specs/wip/`.
- Move implementation-ready specs into `specs/ready/`.
- After implementation is complete, move specs into `specs/done/`.
